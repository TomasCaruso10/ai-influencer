# Research — Pipeline Orchestration + Compliance 2026

## TEMA A — Pipeline Orchestration

### 1. pydantic-graph (Pydantic)

State machine pura, async, dataclass-based. Nodos `BaseNode`, edges salen del type hints del `run`. 3 implementaciones de persistencia (memoria, file, custom — MongoDB si lo implementás).

- **Pros**: cero overhead, Python-native, snapshots por nodo, HITL natural via interrupciones, integración perfecta pydantic-ai. Ideal FSMs con LLMs.
- **Cons**: NO trae scheduler, dashboard, UI. NO retries automáticos ni métricas. Vos corrés el runner loop y agendás jobs.
- **Fit AI influencer**: **ALTO** si pipeline es lineal-con-bifurcaciones (prompt → gen → safety → curar HITL → watermark → publicar). 50-100 piezas/sem son ~14/día — un proceso simple alcanza.

### 2. Prefect 3

Wrappea funciones con `@flow`/`@task`. Dynamic Python-native, sin DAGs rígidos. Cloud $0.005/min serverless (mitad del costo Dagster+). Self-host gratis. Dashboard, scheduling, retries built-in. **Pausas/resume con `pause_flow_run`** — HITL real out-of-the-box.

- **Fit**: **ALTO**. UI para runs, programar batches, reintentar fallos sin código. Curva suave.

### 3. Dagster

**Software-defined assets** — modela tablas/datasets/modelos como assets con lineage. Brilla en data engineering. Pricing 2026 (mayo): saca créditos free Solo/Starter, $0.035-$0.040/crédito.

- **Fit**: BAJO-MEDIO. Pipeline no es asset-heavy. Overkill.

### 4. Airflow

DAG-based clásico. Mucho overhead, scheduler centralizado, no Python-first.

- **Fit**: BAJO. Pesado para 100 piezas/semana.

### 5. Modal

Serverless Python con GPUs. **A10G $1.10/h, H100 SXM $4.29/h** + multipliers regionales (1.25-2.5x) y non-preemption (3x) — puede subir 3.75x. Cold starts 2-4s. NO es orchestrator: es runtime. Combinarlo con Prefect/pydantic-graph.

- **Fit**: **COMPLEMENTARIO**. Execution layer para nodos GPU-heavy.

### 6. Temporal

Workflows durables que sobreviven crashes meses. **Marzo 2026 GA del binding OpenAI Agents SDK ↔ Temporal Python**. Estado en event history; resume exact-once.

- **Cons**: verbosidad alta, modelo cliente/worker, mapping awkward para flows dinámicos con LLMs.
- **Fit**: MEDIO. Overkill para 100 piezas/sem. Si escalás 10x y querés zero data loss: opción más bombproof.

### 7. Argo Workflows

K8s-native, YAML-based. Solo si ya operás K8s.

- **Fit**: BAJO.

### Recomendación

**Stack híbrido**: **pydantic-graph (FSM por pieza) + Prefect 3 (scheduling, retries, dashboard, batches) + Modal (GPU bursts SD/Flux)**.

Razón: ya conocés pydantic-graph, perfecto para lógica por pieza (HITL via interrupciones). Prefect 3 da panel de control y cron/batch sin acoplar a Dagster asset model. Modal para GPU on-demand sin mantener instance 24/7. Costo operacional estimado: <$200/mes para 100 piezas/sem usando A10G ~30s/img.

Si en 6-12 meses crece a 1000+ piezas/sem y empezás a tener fallos por API rate limits o crashes prolongados, migrá outer loop a **Temporal** manteniendo pydantic-graph adentro.

## TEMA B — Compliance 2026 AI Adult Content

### 1. EU AI Act Artículo 50 (vigente 2 agosto 2026)

**Obligaciones**:
- **Providers** (constructor sistema): outputs marcados machine-readable (watermark/metadata), efectivos, interoperables, robustos.
- **Deployers** (usa sistema, publica): **disclosure visible** cuando output es deepfake. Excepción: artísticos/satíricos/ficcionales evidentes — AI adult comercial NO califica.

**Multas**: hasta **€15M o 3% facturación mundial** (lo mayor). SMEs: lo menor. Información falsa al regulador: €7.5M / 1%.

**Scope extraterritorial**: SÍ aplica a empresas fuera EU si output usado por usuarios EU. Necesitás representante legal EU. Aunque seas argentino vendiendo en Fanvue, si tenés subs EU, te aplica.

**Cómo cumplir concretamente**:
- Watermark invisible (SynthID-style o equivalente) + metadata C2PA firmada + texto visible "Generated by AI" en bio/caption.
- Draft Code of Practice (enero 2026, final junio 2026) define detalles técnicos.

### 2. C2PA Content Credentials + SynthID

**C2PA**: metadata firmada criptográficamente que viaja con archivo (manifest: tool, timestamp, edits, creator). Open standard. Meta lo lee desde 2024 — labeliza posts con manifest C2PA reconocido (Firefly, DALL·E 3, Designer). TikTok integró C2PA enero 2025 → **>1.300M videos etiquetados**. Cobertura real **25-30%** (mayoría tools no firma, re-encoding/screenshot mata manifest).

**SynthID** (Google DeepMind): watermark **imperceptible embebido en píxeles**. Sobrevive screenshot, crop, compresión, re-upload. Detection accuracy **95-99%**. Cons: proprietary Google, no interoperable.

**Best practice 2026**: aplicar **ambos** — C2PA para provenance + SynthID/watermark visual para resilience.

**Python SDK** [`c2pa-python`](https://github.com/contentauth/c2pa-python):

```python
import c2pa
manifest = {
    "claim_generator": "ai-influencer/1.0",
    "title": "render_001.jpg",
    "assertions": [
        {"label": "c2pa.actions", "data": {"actions": [
            {"action": "c2pa.created", "softwareAgent": "FluxDev"}
        ]}},
        {"label": "c2pa.training-mining", "data": {"entries": {
            "c2pa.ai_generative_training": {"use": "notAllowed"}
        }}}
    ]
}
builder = c2pa.Builder(manifest)
signer = c2pa.create_signer(callback, c2pa.SigningAlg.PS256, certs, "http://timestamp.digicert.com")
builder.sign_file(signer, "input.jpg", "output.jpg")
```

Prod: usá HSM/KMS para clave privada — no la dejes en disco.

### 3. TAKE IT DOWN Act (US, firma 19 mayo 2025)

**Qué prohíbe**: publicación imágenes íntimas no consensuadas, **incluye deepfakes** y "digital forgeries" de personas identificables reales. **NO aplica a personas 100% sintéticas/ficticias** — el statute exige "intimate visual depiction" sea "of an identifiable individual" reconocible. Pero ojo:

- Personaje sintético se parece demasiado a persona real → riesgo demanda
- LoRA entrenado con fotos persona real sin consentimiento → cae bajo la ley

**Penalidades**:
- Hasta **2 años cárcel** adultos
- **3 años** si involucra minor
- **18 meses** threat-related
- **30 meses** si threat involves minor

**Plataformas**: ventana **48hs takedown** desde notificación. Compliance obligatorio desde **19 mayo 2026**. Primera condena: Columbus, 100+ modelos, 24 plataformas, 700+ imgs — guilty plea.

### 4. AIG-CSAM Safeguards Técnicos

**Federal US law**: AIG-CSAM es ilegal aunque sea **100% sintético, sin víctima real**.

**Pipeline mínimo**:
1. **Prompt-side filter** (pre-gen): regex/embeddings que bloqueen keywords de minor + sexual
2. **Post-gen age estimation**: face detection (MediaPipe/InsightFace) → si hay cara, correr classifier
   - **`dima806/fairface_age_image_detection`** (ViT, FairFace, age buckets) — top descargas mensuales HF
   - **`nateraw/vit-age-classifier`** (85.8M params, FairFace) — bueno documentado
3. **Threshold**: predicción "0-2", "3-9", "10-19" → REJECT siempre. "20-29" + alta confianza → APPROVE. Cualquier ambigüedad → cuarentena HITL.
4. **Sin cara visible**: corré además **NudeNet** o **NSFW classifier Falconsai** + heurísticas proporciones (head:body ratio infantil). Thorn advierte: age models requieren cara, mucho CSAM no la tiene.
5. **Hash matching** contra PhotoDNA/Thorn Safer si publicás gran escala (Thorn licencia SDK)
6. **Block list LoRAs/embeddings**: nunca cargues con "child", "loli", "young teen" en nombre o metadata

Compromiso público Anthropic/OpenAI/Stability/Meta al framework **Thorn "Safety by Design"** — alinearse con eso es baseline industrial.

### 5. NSFW Classifier Post-gen

- **`stabilityai/stable-diffusion-safety-checker`** (CLIP-based): compara contra 17 conceptos sensibles. **False positive rate ~7.96%**
- **LAION CLIP-NSFW Detector**: open, head sobre OpenCLIP ViT-L/14. Mejor balance
- **Ensemble LAION + Detoxify**: reduce 2.6% más false positives

Para AI adult **querés NSFW positivo** (feature). Filtrar: contenido **prohibido** (minors, gore, bestiality, non-consensual). Clasificador multi-clase:
- `Falconsai/nsfw_image_detection` (binary safe/nsfw, para gating)
- **Q16 classifier** (inappropriate concepts: violence, blood, self-harm)
- + age classifier obligatorio

### 7. Argentina — AFIP / ARCA

- **Exportación de servicios**: factura E, IVA 0% si uso económico ocurre exterior. RG 5711 hasta 31 marzo 2026 para acceder al plan
- **Monotributo**: viable hasta tope facturación (cat. K ~$82M anuales 2026, ajustable). Exportación servicios elegible
- **Crypto payouts**: **NO existe prohibición**. AFIP/ARCA lo trata como **ingreso gravado en Ganancias**. VAT exempt para servicios exterior
- **BCRA**: para servicios exterior tradicionalmente exigía liquidar en MULC (Com. A 6700). Hubo flexibilizaciones post-2024 con régimen libre. Verificar normativa vigente
- **CLAE codes** (ARCA): "producción de contenido audiovisual para redes sociales y streaming" — encuadrá ahí
- **Contenido adulto AI**: Argentina sin ley específica anti-AI-adult. Código Penal art. 128 castiga CSAM real y simulado — aplicable a AIG-CSAM. Más allá, SFW/NSFW adulto sintético no prohibido
- **Lavado**: pagos crypto >USD thresholds sin justificar origen → riesgo UIF. Documentá flujo: contrato/TOS plataforma → wallet → exchange registrado → cuenta bancaria con factura E asociada

### 8. Storage / Audit Compliance

- **GDPR**: personaje no basado en persona real → **no hay personal data**, GDPR no aplica al contenido. PERO sí aplica a datos **usuarios EU** (subs, emails, payment)
- **EDPB Coordinated Enforcement Action (19 mar 2026)**: 25 DPAs auditarán transparency Articles 12-14. Privacy policy clara
- **Audit trail mandatorio** EU AI Act:
  - Prompt usado (texto)
  - Modelo + versión + seed + sampler + steps + LoRAs
  - Timestamp gen
  - Safety classifier scores (NSFW, age, Q16) con thresholds
  - Outcome (approved/rejected/quarantined)
  - Reviewer humano si HITL (user_id, decisión)
- **Retención**: mínimo **6 años** (GDPR-records + AI Act technical documentation). Logs sin PII pueden retenerse indefinido. Raw outputs descartados con potencial AIG-CSAM → destruir inmediatamente + reportar (NCMEC US, INHOPE/IWF EU)
- **Backups**: encripcia at-rest (AES-256), separá bucket outputs publicados vs discards vs logs. Versioning + immutable retention para audit logs (S3 Object Lock)
